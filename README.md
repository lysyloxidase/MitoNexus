# MitoNexus

MitoNexus is an AI/ML platform for personalized mitochondrial health analysis. It combines
structured biomarker intake, a LangGraph-based multi-agent reasoning workflow, biomedical
literature retrieval, 3D visualization, and PDF report generation in a single monorepo.

```text
Frontend (Next.js 15)
  -> blood test intake
  -> report UI
  -> 3D graph + mitochondrion views

        |
        v

Backend API (FastAPI)
  -> patient + blood test persistence
  -> report endpoints
  -> PDF download

        |
        v

Agent Runtime (LangGraph + Celery)
  -> biomarker analysis
  -> literature retrieval
  -> therapy reasoning
  -> synthesis

        |
        v

Storage + Services
  -> Postgres + pgvector
  -> Neo4j
  -> Redis
  -> Ollama
  -> Langfuse
  -> public biomedical APIs
  -> WeasyPrint
```

## What Ships In v1.0.0

- FastAPI backend with SQLAlchemy 2 async, Alembic, pgvector, Neo4j, Redis, and Celery
- LangGraph multi-agent workflow with supervisor, biomarker, literature, therapy, and synthesis agents
- 41-marker analysis engine with derived calculations and mitochondrial cascade mappings
- Literature indexing clients for PubMed, bioRxiv, medRxiv, Europe PMC, ClinicalTrials.gov, and Semantic Scholar
- Next.js 15 frontend with:
  - blood test intake form
  - report overview
  - 3D knowledge graph
  - 3D mitochondrion overlay
- PDF report generation via Jinja2 + WeasyPrint
- Langfuse-ready observability hooks

## Quick Start

Five commands are enough to get the stack moving on a clean machine:

```bash
cp .env.example .env
docker compose up -d --build
uv run --directory apps/backend alembic upgrade head
uv run --directory apps/backend uvicorn mitonexus.main:app --reload
pnpm --filter @mitonexus/frontend dev
```

Optional end-to-end demo:

```bash
uv run --directory apps/backend python scripts/demo.py
```

## Hardware Requirements

| Profile | Recommended | Notes |
| --- | --- | --- |
| CPU-only development | 8 cores / 16 GB RAM | Suitable for API, frontend, tests, and non-LLM work |
| Local model inference | NVIDIA GPU with 16 GB+ VRAM | Recommended for `qwen2.5:14b` and `deepseek-r1:14b` |
| Storage | 30 GB+ free disk | Needed for Docker images, databases, Ollama models, and report artifacts |
| Browser / UI | Modern Chromium or Firefox | Required for the 3D graph and mitochondrion views |

## Monorepo Layout

- `apps/backend`: FastAPI API, agents, database models, migrations, tasks, templates, and demo script
- `apps/frontend`: Next.js application, report UI, 3D scenes, and frontend tests
- `packages/shared-types`: shared TypeScript contracts for API and visualization payloads
- `infrastructure`: Docker assets and local helper scripts
- `docs`: architecture, API, development, deployment, and user guidance

## Differentiators

| MitoNexus | Why it matters |
| --- | --- |
| Marker-level mitochondrial mapping | Every supported marker carries cascade, gene, pathway, and interpretation metadata |
| Agentic reasoning workflow | The report is generated by coordinated specialists rather than a single prompt |
| Dual 3D outputs | Users can inspect both a systems graph and a structural mitochondrion overlay |
| Durable async execution | Celery + LangGraph persistence keeps long-running analysis jobs recoverable |
| PDF + UI parity | The same structured report payload drives the web UI and exported PDF |
